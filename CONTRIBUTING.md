# Contributing to FISE Web

Thank you for your interest in contributing to FISE!

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Follow the [manual setup](README.md) instructions for a development environment
4. Create a new branch for your feature or fix

## Development Workflow

1. Make your changes
2. Test your changes locally
3. Ensure your code follows the existing style
4. Commit your changes with a clear message
5. Push to your fork
6. Open a Pull Request

## Pull Request Guidelines

- Keep PRs focused on a single feature or fix
- Update documentation if needed
- Test your changes thoroughly
- Write clear commit messages

## Reporting Issues

When reporting issues, please include:

- Steps to reproduce the problem
- Expected behavior
- Actual behavior
- Environment details (Linux distribution, Linux Version, versions of the prerequisites described in [README](README.md))

## Code Style

- Follow existing patterns in the codebase
- Keep components focused and reusable
- Make sure to correctly format the code with clang-format (eg. clang-format -i ./main.c)

## Questions?

Feel free to open an issue for any questions about contributing.
